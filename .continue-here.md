<decisions_made>

- Option A (incremental schema additions) chosen over full redesign
- costPerHead is always computed as totalCost/splitCount, not stored separately
- amountPaid=null means unpaid; amountPaid=number means paid that amount (replaces boolean `paid`)
- balance = SUM(COALESCE(amountPaid,0) - amount) — negative = owes, positive = credit
- Manual guests use userId prefix `manual-${Date.now()}`
- GOING-only guests get payment records created on sync
- Cost presets stored as JSON in the settings table under key "costPresets"
- Needs follow-up threshold: hardcoded at -$10
- Streaks computed from all runs in DB (not just attended), so a missed run breaks the streak
</decisions_made>
